4. COMPREHENSIVE PROJECT OVERVIEW & SYSTEM ARCHITECTURE
====================================================================================================
End-to-End Architecture for Real-Time Safety:
1. Telemetry Ingestion -> Ingests live IoT and CV data from camera streams and sensors.
2. Golden Path Engine -> State machine monitoring permit conflicts against the environment.
3. Orchestrator Engine -> Evaluates risk scores and triggers escalations/compound risks.
4. LLM-based Action Agent -> Generates corrective actions and regulatory reports via RAG.
5. Safety Dashboard -> Real-time Next.js interface for triage & alerts.

4.1 Frontend Architecture (Next.js App Router)
- Framework: Next.js 16 utilizing Server-Side Rendering (SSR) and Client Components (`"use client"`).
- State Management: React Hooks (`useState`, `useEffect`, `useRef`) and WebSockets for zero-latency bidirectional communication.
- WebGL & Canvas: Custom components render high-performance geospatial heatmaps and live YOLO bounding boxes natively in the browser.
- Styling: Tailwind CSS configured for a premium `#010101` (black) and `#4ade80` (green) HUD aesthetic.

4.2 Backend Architecture (FastAPI Python)
- Framework: FastAPI running on Uvicorn (ASGI) for asynchronous microservices.
- Compound Risk Engine (`risk_engine.py`): Multi-variate correlation engine evaluating simultaneous operations (SimOps).
- Orchestrator (`orchestrator.py`): Event-driven system listening to risk thresholds.
- Vision Model (`vision_model.py`): Ultralytics YOLOv8 inference engine with CPU fallbacks.

4.3 Database Integration & State Management
- TimescaleDB/SQLite: Persistent state of incidents, sensors, and warnings.
- Neo4j Graph Database: Maps complex relationships between Workers, Permits, and Zones.
- Vector Database (Qdrant/Chroma): Stores historical incident reports for our RAG-powered chatbot.

6. MATHEMATICAL EQUATIONS & TECHNOLOGY LOGIC
====================================================================================================

6.1 Compound Risk Calculation (Risk Engine)
The Compound Risk Score (R_c) evaluates simultaneous operations and is calculated using a weighted non-linear probability function:
R_c = 1 - ∏ (1 - w_i * p_i)
Where:
- p_i is the normalized probability/severity of a specific risk factor (e.g., Gas level, Permit proximity).
- w_i is the critical weight assigned to that factor.
Explanation: This mathematical model ensures that multiple moderate risks exponentially amplify the overall compound risk. While a single low-level warning might be ignored by a legacy system, this equation mathematically proves that stacked minor risks create a critical situation, triggering alerts when siloed systems would remain silent.

6.2 Intersection over Union (IoU) for Object Detection
IoU = (Area of Overlap) / (Area of Union)
Explanation: Used by our YOLOv8 Edge AI model to suppress duplicate bounding boxes (Non-Maximum Suppression). If IoU(Box_A, Box_B) > 0.45, the model evaluates that both boxes likely represent the same object (e.g., the same safety helmet). The box with the lower confidence score is mathematically discarded to prevent duplicate counting.

6.3 Mean Average Precision (mAP@0.5)
mAP = (1 / |Classes|) * Σ (Area under Precision-Recall curve for class i)
Explanation: This is the primary metric evaluating the accuracy of our Computer Vision PPE detection model. It calculates the average precision across all object classes (helmets, vests, goggles) at an IoU threshold of 0.5, ensuring our model is both precise (low false positives) and robust (high recall).

6.4 Focal Loss (Handling Class Imbalance)
FL(p_t) = -α_t(1 - p_t)^γ * log(p_t)
Where γ is the focusing parameter (set to 2.0).
Explanation: In safety environments, most workers comply with PPE rules, making violations rare. Standard loss functions get overwhelmed by the majority class. Focal loss dynamically scales down the weight of easily classified examples (compliant workers) and heavily penalizes the model when it misses a rare safety violation (e.g., a missing hard hat).

6.5 Exponential Loss Decay (AdamW Optimization)
L(e) = L_0 * e^(-λ * e) + N(0, σ)
Where λ is the decay rate (0.15) and N(0, σ) injects a dynamically decreasing Gaussian noise jitter.
Explanation: During the simulated neural network backpropagation for our custom vision models, the loss minimizes over time (epochs). This equation allows the learning rate to decay exponentially, while the Gaussian noise helps the optimizer escape local minima, ensuring the model converges to the most optimal weights for real-time inference.

11. HOW TO RUN ON A NEW DEVICE (STEP-BY-STEP DEPLOYMENT GUIDE)
====================================================================================================
To run this project on a completely new laptop or server, you must install the prerequisites and run the environment exactly as described below.

PREREQUISITES (DOWNLOAD THESE FIRST):
1. Python 3.10 or higher (CRITICAL: Ensure "Add Python to PATH" is checked during installation). Download: python.org
2. Node.js (v18 or v20 LTS). Download: nodejs.org
3. Visual Studio Code (Recommended IDE). Download: code.visualstudio.com

STEP 1: OPEN THE PROJECT
--------------------------------------------------
1. Transfer the project folder `hackathon1` to the new device and extract it.
2. Open the folder in Visual Studio Code.
3. Open a New Terminal in VS Code (Terminal -> New Terminal).

STEP 2: SET UP THE PYTHON BACKEND (AI & FASTAPI)
--------------------------------------------------
1. In the terminal, navigate to the root directory if you aren't already:
   cd hackathon1

2. Create a Python Virtual Environment:
   python -m venv .venv
   
3. Activate the Virtual Environment:
   - On Windows (PowerShell): .\ .venv\Scripts\Activate.ps1
   - On Mac/Linux: source .venv/bin/activate
   
4. Install all required Python libraries:
   pip install fastapi uvicorn sqlalchemy pydantic ultralytics opencv-python-headless numpy python-multipart websockets requests

5. Run the Backend Server (with SQLite database fallback enabled):
   - On Windows (PowerShell):
     $env:USE_SQLITE="1"
     python -m uvicorn backend.app.main:app --reload --port 8000
   - On Mac/Linux:
     export USE_SQLITE="1" && python -m uvicorn backend.app.main:app --reload --port 8000

-> The Backend API is now running at http://localhost:8000. KEEP THIS TERMINAL WINDOW OPEN.

STEP 3: SET UP THE NEXT.JS FRONTEND (UI)
--------------------------------------------------
1. Open a SECOND terminal window in VS Code (click the '+' icon in the terminal panel).
2. Navigate to the frontend folder:
   cd frontend
   
3. Install all Node modules (this downloads React, Next.js, Tailwind, etc.):
   npm install
   
4. Start the Frontend Development Server:
   npm run dev

-> The Frontend UI is now running at http://localhost:3000. KEEP THIS TERMINAL WINDOW OPEN.

STEP 4: ACCESS THE PLATFORM
--------------------------------------------------
Open your web browser (Chrome, Edge, or Safari) and go to:
http://localhost:3000

The entire system, including the Compound Risk Engine, Live YOLO Camera Inference, Geospatial Heatmap, and Emergency Orchestrator, is now fully operational on the new device!
